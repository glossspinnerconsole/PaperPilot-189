AI Literature Review Agent

把一句调研需求，变成可审计的双语文献综述。

PaperPilot 面向 AI、AI for Science 和交叉学科研究场景，自动完成需求理解、多源检索、论文筛选、代码仓库发现、开放 PDF 下载、证据综合，并生成中文和英文 Markdown、HTML、PDF 报告。这个项目主页介绍完整 CLI workflow；在线体验在 Cloudflare Workers 提供轻量检索入口。

Why PaperPilot

不是论文清单，而是一条完整的研究工作流。

它把 LLM 的理解和写作能力放在需要判断的地方，把检索、去重、下载、状态记录和输出渲染交给确定性代码。

1

自然语言入口

输入中文或英文调研需求，自动解析主题、年份、代码要求、PDF 下载策略和多样化检索关键词。

2

证据驱动报告

报告从 canonical model 渲染，论文编号、参考文献、代码链接和中英文结论保持一致。

3

可观察运行

每次任务生成独立 run folder，保存 state、events、protocol、corpus、verification 和 quality gate。

Agent Workflow

Intake 到 Report 的状态机式 Agent。

默认尽量自动，但关键阶段留下产物和诊断信息，方便复查、恢复和二次分析。

01

Intake

解析用户需求、当前模型、来源配置和任务参数。

02

Protocol

生成研究问题、纳入/排除标准、负面关键词和检索计划。

03

Search

并发调用 Source Registry，也支持导入本地 PDF、BibTeX、RIS。

04

Corpus

统一论文模型，按 DOI、arXiv、PMID、标题相似度去重。

05

Report

默认最多纳入 100 篇文献，生成中英文 Markdown、HTML、PDF，并附引用编号和代码仓库。

06

Wiki

同步输出 Obsidian Wiki：论文页、方法页、主题页、claim 页和可追踪 manifest。

Source Registry

覆盖 AI/CS、生医和 AI for Science 的多源检索。

稳定免 key 来源默认启用；需要 API key 的来源在配置后自动加入检索和 doctor 自检。

Quick Start

几条命令就能开始。

首次运行会生成 `~/.paperpilot/config.json`，你可以配置 OpenAI-compatible LLM 和可选检索来源 API。

python -m pip install paperpilot -i https://pypi.org/simple
